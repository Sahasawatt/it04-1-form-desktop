# IT 04-1 Registration Form — Desktop

A standalone Windows desktop rewrite of the IT 04-1 registration form. The
original is a Next.js + Postgres + Docker web app; this version is a single
self-contained `.exe` with a local SQLite database — no server, no Docker,
no separate runtime install.

## Requirements covered

The form collects the 8 IT 04-1 fields:

| Field | Notes |
|---|---|
| First Name | required |
| Last Name | required |
| Email | required, email format |
| Phone | required, phone format |
| Profile | image upload, stored as a Base64 data URL |
| Birth Day | calendar picker, displayed/entered as dd/MM/yyyy |
| Occupation | combo box, mock data list |
| Sex | Male / Female |

Every field is validated before save.

- **Save** — inserts the record (the database generates the ID via
  `AUTOINCREMENT`), shows `save data success !!!  ID: N`, then clears the
  form.
- **Clear** — resets the form.
- **Saved Records** — a separate window listing every saved record, with a
  thumbnail per row and the ability to open the uploaded image at full size
  (double-click a row, or select a row and click "View image").

## Run

Double-click `IT04-1Form.exe`. Nothing to install — the .NET runtime and
SQLite are bundled inside the single file.

## Build from source

Requires the .NET 8 SDK.

```
dotnet publish src/IT04Form -c Release
```

The single-file exe is produced at:

```
src\IT04Form\bin\Release\net8.0-windows\win-x64\publish\IT04-1Form.exe
```

No extra CLI flags are needed — the publish settings (self-contained,
single-file, win-x64) already live in `IT04Form.csproj`.

## Database

A local SQLite file, created automatically on first run at:

```
%LOCALAPPDATA%\IT04-1Form\it04.db
```

`persons` table:

| Column | Notes |
|---|---|
| id | `INTEGER PRIMARY KEY AUTOINCREMENT` — generated by the database |
| firstName | |
| lastName | |
| email | |
| phone | |
| birthDay | stored as `yyyy-MM-dd` |
| occupation | |
| sex | |
| profileBase64 | data URL, e.g. `data:image/png;base64,...` |
| createdAt | `yyyy-MM-dd HH:mm:ss`, UTC |

Deleting `it04.db` resets the app to an empty state — it is recreated
automatically the next time the app starts.

## Project structure

```
IT04Form.sln
src/IT04Form/
├── IT04Form.csproj
├── Program.cs          entry point; initializes the DB, runs MainForm
├── Models/
│   ├── Person.cs       a row of the persons table
│   └── PersonInput.cs  raw form values, pre-validation
├── Core/
│   ├── Constants.cs    occupation list, sex options, messages, field keys
│   └── Validation.cs   field validation rules
├── Data/
│   └── Db.cs           SQLite access (init, insert, list)
├── Imaging/
│   └── ImageUtil.cs    Base64 data URL ⇄ Image conversion
└── UI/
    ├── MainForm.cs     the registration form (8 fields, validation, Save/Clear)
    └── RecordsForm.cs  the Saved Records window
```
